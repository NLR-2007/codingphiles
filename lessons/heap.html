Heap Memory in Java

Heap Memory is a runtime memory area where objects and instance variables are stored.

Heap memory is created when JVM starts and is shared by all threads.

Why do we need Heap Memory?

Heap memory is needed to store objects dynamically during program execution.

Without heap memory:
✔ Objects cannot be created
✔ Data cannot be shared between methods

How Objects are Stored

Whenever we use the new keyword, memory is allocated in heap.

Example:
Student s = new Student();

✔ Object → Heap Memory
✔ Reference variable → Stack Memory

Heap Memory Features

✔ Stores objects
✔ Large memory
✔ Slower than stack
✔ Managed by Garbage Collector

Heap Memory (Easy to Remember)

Heap memory stores objects and lives till JVM runs.

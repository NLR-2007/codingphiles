Synchronization in Java

Synchronization in Java is used to control access to shared resources in a multithreaded environment.

It prevents data inconsistency and ensures thread safety.

Why Synchronization is Needed?

When multiple threads access the same resource simultaneously, it may lead to:

✔ Data inconsistency
✔ Race condition
✔ Incorrect output

Synchronization ensures that only one thread accesses the shared resource at a time.

What is Race Condition?

A race condition occurs when multiple threads try to modify shared data at the same time.

Result depends on execution order of threads, which is unpredictable.

Example Without Synchronization

class Counter {
  int count = 0;

  void increment() {
    count++;
  }
}

class Test {
  public static void main(String[] args) {
    Counter c = new Counter();
    // Multiple threads accessing increment()
  }
}

❌ Output may be incorrect due to race condition

What is Synchronization?

Synchronization is the process of locking an object or method so that only one thread can execute it at a time.

Java provides synchronization using:

✔ synchronized method
✔ synchronized block

Synchronized Method

A synchronized method locks the object for the entire method.

Syntax

synchronized returnType methodName() {
  // critical section
}

Example

class Counter {
  int count = 0;

  synchronized void increment() {
    count++;
  }
}

✔ Only one thread executes method at a time

Synchronized Block

A synchronized block locks only a specific section of code.

It provides better performance than synchronized methods.

Syntax

synchronized(object) {
  // critical section
}

Example

class Counter {
  int count = 0;

  void increment() {
    synchronized(this) {
      count++;
    }
  }
}

Object Lock vs Class Lock

✔ Object lock → synchronized instance method / block
✔ Class lock → synchronized static method

Static Synchronization

static synchronized void show() {
  // class-level lock
}

Advantages of Synchronization

✔ Prevents race condition
✔ Ensures data consistency
✔ Provides thread safety

Disadvantages of Synchronization

✔ Performance overhead
✔ Reduced concurrency
✔ Possibility of deadlock

Synchronization vs Non-Synchronization

✔ Synchronized → Safe but slower
✔ Non-synchronized → Fast but unsafe

Best Practices

✔ Synchronize only critical sections
✔ Prefer synchronized blocks over methods
✔ Avoid nested synchronization
✔ Do not synchronize unnecessary code

Real-Time Examples

✔ Bank account transactions
✔ Seat booking systems
✔ Shared counters
✔ Inventory management

Common Mistakes

✔ Synchronizing entire method unnecessarily
✔ Ignoring synchronization for shared data
✔ Assuming synchronization improves performance

Synchronization (Interview / Exam Ready)

Synchronization in Java is a mechanism used to control access to shared resources in a multithreaded environment. It prevents race conditions and ensures data consistency by allowing only one thread to execute critical sections of code at a time. Java supports synchronization using synchronized methods and synchronized blocks.

Thread Priority in Java

Thread Priority in Java is used to suggest the order of execution of threads to the Thread Scheduler.

Threads with higher priority are generally executed before threads with lower priority, but this behavior depends on the JVM and OS.

What is Thread Priority?

Thread priority is an integer value associated with each thread that indicates its importance.

✔ Higher priority → Higher chance of execution
✔ Lower priority → Lower chance of execution

⚠ Priority does NOT guarantee execution order

Priority Range in Java

Java defines thread priorities in the range:

✔ MIN_PRIORITY = 1
✔ NORM_PRIORITY = 5 (default)
✔ MAX_PRIORITY = 10

Default Thread Priority

If no priority is set, a thread inherits the priority of the thread that created it.

✔ Default priority = 5

Getting Thread Priority

The getPriority() method is used to retrieve thread priority.

Example:

Thread t = Thread.currentThread();
int p = t.getPriority();

Setting Thread Priority

The setPriority() method is used to change thread priority.

Example:

Thread t = new Thread();
t.setPriority(Thread.MAX_PRIORITY);

⚠ Value must be between 1 and 10
Otherwise → IllegalArgumentException

Example: Thread Priority

class MyThread extends Thread {
  public void run() {
    System.out.println(Thread.currentThread().getName());
  }
}

class Test {
  public static void main(String[] args) {
    MyThread t1 = new MyThread();
    MyThread t2 = new MyThread();

    t1.setPriority(Thread.MAX_PRIORITY);
    t2.setPriority(Thread.MIN_PRIORITY);

    t1.start();
    t2.start();
  }
}

Important Points

✔ Priority is only a hint to scheduler
✔ JVM and OS decide actual execution
✔ Do not rely on priority for program logic

Thread Priority vs Scheduling

✔ Priority affects scheduling preference
✔ Scheduling algorithm is platform dependent
✔ Same priority threads may execute randomly

Real‑Time Usage

✔ UI threads → Higher priority
✔ Background logging → Lower priority
✔ Monitoring services → Medium priority

Common Mistakes

✔ Assuming priority guarantees execution order
✔ Using priority instead of synchronization
✔ Setting invalid priority values

StringBuilder in Java

StringBuilder is a class in Java used to create mutable (modifiable) strings.

It is similar to StringBuffer, but it is not thread-safe and provides better performance.

Package of StringBuilder

StringBuilder belongs to the java.lang package, so it is available by default.

Why StringBuilder is Needed?

The String class is immutable, which means every modification creates a new object.

StringBuilder allows modification of the same object, making it efficient for frequent changes.

Key Features of StringBuilder

✔ Mutable (content can be changed)
✔ Not thread-safe
✔ Faster than String and StringBuffer

Creating a StringBuilder Object

Example:

StringBuilder sb = new StringBuilder("Java");

Modifying StringBuilder

Example:

StringBuilder sb = new StringBuilder("Java");
sb.append(" Programming");
System.out.println(sb);

Output:
Java Programming

Commonly Used StringBuilder Methods

1. append()

Adds text at the end of the string.

Example:

sb.append(" World");

2. insert()

Inserts text at a specified index.

Example:

sb.insert(4, " Language");

3. replace()

Replaces characters between given indexes.

Example:

sb.replace(0, 4, "Core");

4. delete()

Deletes characters between specified indexes.

Example:

sb.delete(0, 4);

5. reverse()

Reverses the content of the string.

Example:

sb.reverse();

Capacity of StringBuilder

StringBuilder has an internal capacity that increases automatically when required.

Default capacity = 16 + initial string length

Example:

StringBuilder sb = new StringBuilder();
System.out.println(sb.capacity());

Thread Safety

StringBuilder is not synchronized, which means it is not thread-safe.

Because of this, it performs faster than StringBuffer.

Performance Comparison

✔ String → Slow (immutable)
✔ StringBuffer → Medium (thread-safe)
✔ StringBuilder → Fast (non-thread-safe)

When to Use StringBuilder?

Use StringBuilder when:
✔ String is frequently modified
✔ Application is single-threaded
✔ Performance is critical

StringBuilder vs StringBuffer

✔ StringBuilder → Faster, not thread-safe
✔ StringBuffer → Slower, thread-safe

Limitations of StringBuilder

✔ Not safe for multi-threaded environments
✔ Requires external synchronization if used by multiple threads

Real-Time Use Cases

StringBuilder is used in:
✔ Building dynamic strings
✔ Loop-based string concatenation
✔ Data formatting
✔ Performance-critical applications

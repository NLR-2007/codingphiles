Thread Methods in Java

The Thread class provides several methods to control the execution, behavior, and life cycle of threads.

These methods are essential for multithreading control.

Commonly Used Thread Methods

✔ start()
✔ run()
✔ sleep()
✔ join()
✔ yield()
✔ setName()
✔ getName()
✔ setPriority()
✔ getPriority()
✔ isAlive()
✔ currentThread()

start()

The start() method begins execution of a new thread.

✔ Creates a new thread
✔ Internally calls run()

t.start();

⚠ start() can be called only once

run()

The run() method contains the code executed by the thread.

✔ Executed by JVM when start() is called
✔ Calling run() directly does NOT create new thread

sleep()

The sleep() method pauses the current thread for a specified time.

Syntax:

Thread.sleep(milliseconds);

Example

Thread.sleep(1000); // pauses for 1 second

✔ Throws InterruptedException
✔ Thread goes to Blocked state

join()

The join() method makes one thread wait for another thread to finish.

Example

t1.join(); // main thread waits for t1

✔ Used for dependency control
✔ Throws InterruptedException

yield()

The yield() method pauses the current thread and gives chance to other threads of same priority.

Example:

Thread.yield();

✔ Suggestion to scheduler
✔ No guarantee of execution order

setName() and getName()

Used to set or get the name of a thread.

t.setName("WorkerThread");
String name = t.getName();

setPriority() and getPriority()

Thread priority determines execution preference.

Priority range:
✔ MIN_PRIORITY = 1
✔ NORM_PRIORITY = 5
✔ MAX_PRIORITY = 10

t.setPriority(Thread.MAX_PRIORITY);

⚠ Priority depends on OS and JVM

isAlive()

Checks whether a thread is alive.

✔ true → thread is running or ready
✔ false → thread is terminated

t.isAlive();

currentThread()

Returns reference to the currently executing thread.

Thread t = Thread.currentThread();

Deprecated Thread Methods

The following methods are deprecated due to security issues:

✖ stop()
✖ suspend()
✖ resume()

⚠ Do NOT use these methods

Thread Methods and Life Cycle

✔ start() → New → Runnable
✔ sleep() → Running → Blocked
✔ join() → Running → Waiting
✔ run() end → Terminated

Real‑Time Examples

✔ sleep() → Game animation delay
✔ join() → Wait for file download
✔ yield() → CPU sharing
✔ isAlive() → Thread monitoring

Common Mistakes

✔ Calling run() instead of start()
✔ Ignoring InterruptedException
✔ Assuming thread priority guarantees order

Thread Methods (Interview / Exam Ready)

Thread methods in Java control thread execution and behavior. Common methods include start(), run(), sleep(), join(), yield(), isAlive(), and currentThread(). These methods help manage thread life cycle, synchronization, and execution flow in multithreaded applications.

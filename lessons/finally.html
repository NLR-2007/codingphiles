finally Block in Java

The finally block in Java is used to execute important code whether an exception occurs or not.

It is mainly used for resource cleanup.

What is finally Block?

The finally block is associated with a try-catch block.

✔ Executes always
✔ Runs after try or catch
✔ Used for cleanup code

Why Do We Need finally?

The finally block is needed to:

✔ Close files
✔ Close database connections
✔ Release resources
✔ Maintain program reliability

Syntax of finally

try {
  // risky code
} catch (Exception e) {
  // exception handling
} finally {
  // cleanup code
}

finally with Exception

class Demo {
  public static void main(String[] args) {
    try {
      int a = 10 / 0;
    } catch (ArithmeticException e) {
      System.out.println("Exception handled");
    } finally {
      System.out.println("Finally block executed");
    }
  }
}

Output:
Exception handled
Finally block executed

finally without Exception

try {
  int a = 10 / 2;
} catch (Exception e) {
  System.out.println("Exception");
} finally {
  System.out.println("Finally always executes");
}

finally without catch

A finally block can exist without a catch block.

try {
  System.out.println("Try block");
} finally {
  System.out.println("Finally block");
}

✔ Used when no exception handling is required
✔ Cleanup is mandatory

Cases When finally Does NOT Execute

Finally block does not execute when:

✔ System.exit() is called
✔ JVM crashes
✔ Power failure occurs

finally and return Statement

If return is present in try or catch, finally executes before returning.

Example:

try {
  return 10;
} finally {
  System.out.println("Finally executed");
}

finally vs try-with-resources

✔ finally → Manual resource cleanup
✔ try-with-resources → Automatic resource cleanup

✔ finally → Older approach
✔ try-with-resources → Recommended (Java 7+)

Real-Time Examples

✔ Closing database connection
✔ Closing file streams
✔ Releasing network resources

Common Mistakes

✔ Writing business logic in finally
✔ Using finally for exception handling
✔ Forgetting finally for resource cleanup

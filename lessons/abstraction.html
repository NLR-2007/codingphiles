Abstraction in Java

Abstraction is one of the core Object-Oriented Programming (OOP) concepts.

Abstraction means hiding implementation details and showing only essential features to the user.

What is Abstraction?

Abstraction focuses on what an object does rather than how it does it.

It helps reduce complexity and improves code maintainability.

Why Do We Need Abstraction?

Abstraction provides:
✔ Security (hide internal logic)
✔ Reduced complexity
✔ Loose coupling
✔ Easy maintenance and scalability

How to Achieve Abstraction in Java?

In Java, abstraction is achieved using:
✔ Abstract Class
✔ Interface

Abstract Class

An abstract class is a class that is declared using the abstract keyword.

Features:
✔ Can have abstract methods (no body)
✔ Can have concrete methods (with body)
✔ Cannot be instantiated

Example of Abstract Class

abstract class Vehicle {
  abstract void start();

  void fuel() {
    System.out.println("Fuel is required");
  }
}

class Car extends Vehicle {
  void start() {
    System.out.println("Car starts with key");
  }
}

Abstract Method

An abstract method is a method that does not have a body.

It must be implemented by child classes.

Example:

abstract void start();

Rules of Abstract Class

✔ Abstract class can have constructors
✔ Abstract class can have static methods
✔ If a class has an abstract method, class must be abstract
✔ Child class must implement all abstract methods

Abstraction Using Interface (Overview)

An interface provides complete abstraction.

It contains abstract methods by default (before Java 8).

Example:

interface Shape {
  void draw();
}

Abstraction vs Encapsulation

✔ Abstraction → Hides implementation details
✔ Encapsulation → Hides data using access modifiers

✔ Abstraction → Focus on design
✔ Encapsulation → Focus on security

Real-Time Examples

✔ ATM → User knows withdraw(), not internal process
✔ Car → User drives, doesn't know engine logic
✔ Mobile → User taps icons, OS handles logic

Advantages of Abstraction

✔ Improves code flexibility
✔ Supports scalability
✔ Enhances security
✔ Reduces code dependency

Common Mistakes

✔ Trying to create object of abstract class
✔ Not implementing abstract methods in child class
✔ Confusing abstraction with encapsulation

Abstraction (Interview / Exam Ready)

Abstraction in Java is an OOP concept that hides implementation details and shows only essential features to the user. It is achieved using abstract classes and interfaces, improving security, flexibility, and maintainability.

Java Architecture

Java Architecture explains how a Java program is compiled, loaded, and executed inside a computer system.

It shows the interaction between:
✔ Java Source Code
✔ Compiler
✔ JVM (Java Virtual Machine)
✔ Operating System
✔ Hardware

Overview of Java Architecture

Java follows a layered architecture that makes it platform independent.

The main components are:
✔ Java Source Code (.java)
✔ Java Compiler (javac)
✔ Bytecode (.class)
✔ JVM (Java Virtual Machine)
✔ Operating System & Hardware

Java Program Execution Flow

The execution of a Java program happens in the following steps:

Step 1: Programmer writes Java source code (.java file)
Step 2: Java compiler converts source code into bytecode (.class file)
Step 3: Bytecode is loaded into JVM
Step 4: JVM converts bytecode into machine code
Step 5: Machine code is executed by the operating system

Java Compiler (javac)

The Java compiler converts Java source code into bytecode.

Functions of Java Compiler:
✔ Checks syntax errors
✔ Converts .java file into .class file
✔ Produces platform-independent bytecode

Java Bytecode

Bytecode is an intermediate code generated by the Java compiler.

Characteristics of Bytecode:
✔ Not machine dependent
✔ Same for all platforms
✔ Executed only by JVM

Bytecode is the main reason for Java’s platform independence.

Java Virtual Machine (JVM)

JVM is the heart of Java architecture.

It is responsible for executing Java bytecode and converting it into machine-specific instructions.

Components of JVM

The JVM consists of the following main components:

✔ Class Loader Subsystem
✔ Runtime Data Areas
✔ Execution Engine
✔ Native Method Interface (JNI)
✔ Native Method Libraries

Class Loader Subsystem

The Class Loader loads .class files into JVM memory.

Types of Class Loaders:
✔ Bootstrap Class Loader
✔ Extension Class Loader
✔ Application Class Loader

It also performs bytecode verification for security.

Runtime Data Areas

Runtime Data Areas are memory areas created during program execution.

They include:
✔ Method Area – Class metadata
✔ Heap Area – Objects and instances
✔ Stack Area – Method calls and local variables
✔ PC Register – Instruction address
✔ Native Method Stack – Native code execution

Execution Engine

The Execution Engine executes bytecode instructions.

It consists of:
✔ Interpreter – Executes bytecode line by line
✔ JIT Compiler – Improves performance by compiling bytecode to machine code
✔ Garbage Collector – Frees unused memory

Garbage Collector

Garbage Collector automatically manages memory in Java.

Functions:
✔ Identifies unused objects
✔ Deletes them from heap memory
✔ Prevents memory leaks

This makes Java memory-safe and developer-friendly.

Java Native Interface (JNI)

JNI allows Java programs to interact with native applications written in C or C++.

It is used when Java needs to access hardware-level features.

JRE and JDK in Java Architecture

JRE (Java Runtime Environment) contains:
✔ JVM
✔ Core libraries

JDK (Java Development Kit) contains:
✔ JRE
✔ Compiler (javac)
✔ Development tools

Java Architecture Diagram Explanation

Java source code is compiled into bytecode by the compiler. The bytecode is loaded into the JVM using the class loader. The execution engine executes the bytecode with help of runtime memory areas and garbage collector. Finally, the machine code runs on the operating system.

Advantages of Java Architecture

✔ Platform independence
✔ High security
✔ Automatic memory management
✔ High performance using JIT
✔ Scalable and robust

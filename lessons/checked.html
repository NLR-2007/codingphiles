Checked Exceptions in Java

Checked Exceptions are exceptions that are checked at compile time by the Java compiler.

If a checked exception occurs, the program must handle it, otherwise the code will not compile.

What are Checked Exceptions?

Checked exceptions represent conditions that a program can anticipate and recover from.

They usually occur due to:
✔ External resources
✔ Input/output operations
✔ Database connectivity
✔ Network communication

Why Are They Called Checked Exceptions?

They are called checked because:

✔ The compiler checks them at compile time
✔ The programmer is forced to handle them

This improves program reliability.

Examples of Checked Exceptions

Common checked exceptions include:

✔ IOException
✔ FileNotFoundException
✔ SQLException
✔ ClassNotFoundException
✔ InterruptedException

Checked Exception Example

import java.io.FileReader;

class Demo {
  public static void main(String[] args) {
    FileReader fr = new FileReader("data.txt");
  }
}

❌ Compile‑time error because exception is not handled

Handling Checked Exceptions

Checked exceptions can be handled in two ways:

✔ Using try‑catch block
✔ Using throws keyword

Handling Using try‑catch

try {
  FileReader fr = new FileReader("data.txt");
} catch (FileNotFoundException e) {
  System.out.println("File not found");
}

✔ Exception handled locally
✔ Program continues normally

Handling Using throws

The throws keyword is used to declare exceptions in the method signature.

Example:

void readFile() throws IOException {
  FileReader fr = new FileReader("data.txt");
}

✔ Responsibility passed to caller method

Checked Exceptions vs Unchecked Exceptions

✔ Checked → Compile‑time checked
✔ Unchecked → Runtime checked

✔ Checked → Must be handled
✔ Unchecked → Not mandatory to handle

✔ Checked → Extend Exception
✔ Unchecked → Extend RuntimeException

Why Java Enforces Checked Exceptions?

Java enforces checked exceptions to:

✔ Force error handling
✔ Improve program robustness
✔ Avoid unexpected program termination

Real-Time Examples

✔ File operations → IOException
✔ Database connection → SQLException
✔ Thread sleep → InterruptedException

Common Mistakes

✔ Forgetting to handle checked exceptions
✔ Catching generic Exception instead of specific one
✔ Ignoring exceptions using empty catch block

Checked Exceptions (Interview / Exam Ready)

Checked exceptions in Java are exceptions that are checked at compile time. They must be either handled using try‑catch blocks or declared using the throws keyword. Examples include IOException, SQLException, and FileNotFoundException. Checked exceptions improve program reliability by enforcing proper error handling.

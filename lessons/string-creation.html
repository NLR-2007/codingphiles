String Object Creation in Java

In Java, a String object can be created in multiple ways.

Understanding how String objects are created is very important for:
✔ Memory management
✔ Performance optimization
✔ Interview questions

Ways to Create String Objects

Java provides two main ways to create String objects:
✔ Using String Literal
✔ Using new Keyword

1. String Object Creation Using String Literal

When a String is created using a literal, it is stored in a special memory area called the String Constant Pool (SCP).

Example:

String s1 = "Java";
String s2 = "Java";

Explanation

✔ JVM checks the String Constant Pool
✔ If "Java" already exists, reference is reused
✔ No new object is created for s2

As a result, both s1 and s2 point to the same String object.

Memory Representation (String Literal)

✔ Only one object is created in SCP
✔ Multiple references can point to it
✔ Memory efficient

2. String Object Creation Using new Keyword

When a String is created using the new keyword, a new object is always created in heap memory.

Example:

String s3 = new String("Java");
String s4 = new String("Java");

Explanation

✔ JVM creates a new object in heap for each statement
✔ Even if the value is same, objects are different
✔ SCP may still contain the literal "Java"

Memory Representation (new Keyword)

✔ New object created in heap every time
✔ Less memory efficient
✔ Useful when separate objects are required

Comparison Example

String a = "Java";
String b = "Java";
String c = new String("Java");

a == b → true (same reference)
a == c → false (different reference)
a.equals(c) → true (same content)

String Interning

String Interning is the process of adding a string to the String Constant Pool.

Example:

String s = new String("Java");
String t = s.intern();

Now, t refers to the SCP object.

Which Method is Better?

✔ Use String Literal for better performance
✔ Use new keyword only when a separate object is needed

Common Mistakes

✔ Using == instead of equals() for comparison
✔ Creating unnecessary String objects
✔ Not understanding SCP behavior

Real-Time Use Cases

String object creation is important in:
✔ Login systems
✔ Configuration values
✔ File paths and URLs
✔ Database queries

String Object Creation (Interview / Exam Ready)

In Java, String objects can be created using string literals (stored in the String Constant Pool) or using the new keyword (stored in heap memory). String literals are memory efficient, while the new keyword always creates a new object.

Immutable Strings in Java

In Java, a String is immutable. Immutable means unchangeable.

Once a String object is created, its value cannot be modified. Any change creates a new String object.

What Does Immutability Mean?

Immutability means:
✔ The content of a String cannot be changed
✔ Any modification creates a new object
✔ The original String remains unchanged

Example of Immutable String

String s = "Java";
s.concat(" Programming");
System.out.println(s);

Output:
Java

Explanation:
✔ concat() creates a new String object
✔ The new object is not assigned to any reference
✔ Original string remains unchanged

Correct Way to Modify a String

String s = "Java";
s = s.concat(" Programming");
System.out.println(s);

Output:
Java Programming

Why Are Strings Immutable in Java?

1. Security

Strings are used in:
✔ Passwords
✔ Database URLs
✔ Network connections

If Strings were mutable, attackers could change sensitive data.

2. String Constant Pool (SCP)

String literals are stored in the String Constant Pool.

Multiple references can point to the same String object.

Immutability ensures that one reference cannot change the value for others.

3. Thread Safety

Because Strings cannot be modified, they are naturally thread-safe.

Multiple threads can safely share the same String object.

4. Performance & Hashing

String hashcode is cached.

Immutability ensures the hashcode does not change, improving performance in collections like HashMap.

Immutable vs Mutable Strings

✔ String → Immutable
✔ StringBuffer → Mutable
✔ StringBuilder → Mutable

When Immutability is a Problem?

Immutability causes performance issues when:
✔ Strings are modified repeatedly
✔ Concatenation happens inside loops

In such cases, use:
✔ StringBuilder (fast)
✔ StringBuffer (thread-safe)

Real-Time Examples

Immutable Strings are useful in:
✔ Login systems
✔ Configuration values
✔ Constants and keys
✔ Multi-threaded applications

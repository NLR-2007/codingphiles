Multiple catch Blocks in Java

Java allows multiple catch blocks with a single try block to handle different types of exceptions separately.

This provides more precise and meaningful exception handling.

What are Multiple catch Blocks?

When a try block can throw more than one type of exception, multiple catch blocks are used to handle each exception type.

Each catch block handles a specific exception.

Syntax of Multiple catch

try {
  // risky code
} catch (ExceptionType1 e) {
  // handling code
} catch (ExceptionType2 e) {
  // handling code
}

Example: Multiple catch Blocks

class Demo {
  public static void main(String[] args) {
    try {
      int[] arr = {1, 2, 3};
      int a = 10 / 0;
      System.out.println(arr[5]);
    } catch (ArithmeticException e) {
      System.out.println("Arithmetic Exception occurred");
    } catch (ArrayIndexOutOfBoundsException e) {
      System.out.println("Array Index Out of Bounds");
    } catch (Exception e) {
      System.out.println("General Exception");
    }
  }
}

Important Rule (Exception Hierarchy Rule)

✔ Child (specific) exception must be caught before parent (generic) exception.

✔ Otherwise, compiler error occurs.

Wrong Order (Compile-Time Error)

try {
  // code
} catch (Exception e) {
  // generic
} catch (ArithmeticException e) {
  // unreachable code
}

Correct Order

try {
  // code
} catch (ArithmeticException e) {
  // specific
} catch (Exception e) {
  // generic
}

Multiple catch vs Single catch

✔ Multiple catch → Specific handling
✔ Single catch → Generic handling

✔ Multiple catch → Better error messages
✔ Single catch → Less control

Java 7: Multi-catch (Brief)

From Java 7 onwards, multiple exceptions can be handled in a single catch block.

Example:

catch (ArithmeticException | NullPointerException e) {
  System.out.println("Exception occurred");
}

Advantages of Multiple catch

✔ Handles different exceptions separately
✔ Improves program readability
✔ Better debugging and maintenance

Real-Time Examples

✔ Banking app → divide by zero, invalid input
✔ File handling → file not found, IO error
✔ Web app → null input, format issues

Common Mistakes

✔ Writing generic catch first
✔ Catching Exception unnecessarily
✔ Duplicating same handling code

Multiple catch (Interview / Exam Ready)

Multiple catch blocks in Java allow handling different types of exceptions separately. Each catch block handles a specific exception, and they must be ordered from most specific to most general to avoid compile-time errors. This improves clarity and robustness of exception handling.

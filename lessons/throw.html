throw Keyword in Java

The throw keyword in Java is used to explicitly create and throw an exception from a method or block of code.

It is mainly used for custom error handling.

What is throw?

The throw keyword is used to:

✔ Create an exception object
✔ Pass it to the JVM
✔ Interrupt normal program flow

Only one exception can be thrown at a time.

Syntax of throw

throw new ExceptionType("Error message");

Simple Example of throw

class Demo {
  public static void main(String[] args) {
    int age = 15;

    if (age < 18) {
      throw new ArithmeticException("Not eligible to vote");
    }
  }
}

throw with try-catch

An exception thrown using throw can be handled using try-catch.

try {
  throw new NullPointerException("Null value");
} catch (NullPointerException e) {
  System.out.println(e.getMessage());
}

throw Checked Exception

If a checked exception is thrown, it must be:

✔ Handled using try-catch
✔ Or declared using throws

Example:

throw new IOException("File not found");

❌ Compile-time error if not handled

throw Unchecked Exception

Unchecked exceptions can be thrown without handling.

Example:

throw new ArithmeticException("Invalid calculation");

throw vs throws

✔ throw → Used to throw an exception
✔ throws → Used to declare exceptions

✔ throw → Used inside method body
✔ throws → Used in method signature

✔ throw → Throws one exception at a time
✔ throws → Can declare multiple exceptions

throw with User-Defined Exception

User-defined exceptions are commonly thrown using the throw keyword.

class InvalidAgeException extends Exception {
  InvalidAgeException(String msg) {
    super(msg);
  }
}

class Test {
  void checkAge(int age) throws InvalidAgeException {
    if (age < 18) {
      throw new InvalidAgeException("Age must be 18 or above");
    }
  }
}

Important Rules of throw

✔ Only Throwable objects can be thrown
✔ Control stops immediately after throw
✔ Code after throw is unreachable

Real-Time Examples

✔ Banking app → insufficient balance
✔ Login system → invalid credentials
✔ Voting system → age validation

Common Mistakes

✔ Writing code after throw statement
✔ Throwing generic Exception unnecessarily
✔ Not handling checked exceptions

throw Keyword (Interview / Exam Ready)

The throw keyword in Java is used to explicitly throw an exception from a method or block of code. It allows developers to create custom error conditions and is commonly used with user-defined exceptions for validation and business logic enforcement.

Constructors in Java

A Constructor is a special method used to initialize objects in Java.

It is automatically called when an object of a class is created.

Key Features of Constructors

✔ Constructor name is same as class name
✔ Constructor has no return type (not even void)
✔ Called automatically during object creation
✔ Used to initialize instance variables

Why Do We Need Constructors?

Constructors are used to:
✔ Initialize object data
✔ Allocate resources
✔ Ensure object is created in a valid state

Example of Constructor

class Student {
  int id;

  Student() {
    id = 101;
  }
}

Student s = new Student();

Here, the constructor initializes the variable id.

Types of Constructors

Java mainly supports:
✔ Default Constructor
✔ Parameterized Constructor

Rules of Constructors

✔ Constructor cannot be static
✔ Constructor cannot be abstract or final
✔ Constructors can be overloaded
✔ Constructor is not inherited

Constructor Overloading

Multiple constructors with different parameters in the same class is called Constructor Overloading.

Example:

class Demo {
  Demo() { }
  Demo(int x) { }
  Demo(int x, int y) { }
}

Constructor vs Method

✔ Constructor → Initializes object
✔ Method → Performs operation

✔ Constructor → Same name as class
✔ Method → Any valid name

✔ Constructor → No return type
✔ Method → Has return type

Real-Time Example

In real life:
✔ Mobile constructor initializes RAM, OS, Storage
✔ Car constructor initializes engine, model, color

Common Mistakes

✔ Writing return type for constructor
✔ Forgetting constructor name must match class
✔ Thinking constructor is optional for initialization

Constructors (Interview / Exam Ready)

A constructor is a special method in Java used to initialize objects. It has the same name as the class, no return type, and is automatically called when an object is created.

protected Access Modifier in Java

The protected access modifier allows class members to be accessed within the same package and by subclasses even if they are in different packages.

What is protected?

The protected access modifier is less restrictive than private and default, but more restrictive than public.

It is mainly used to support inheritance.

Where Can protected Be Used?

The protected access modifier can be applied to:

✔ Variables
✔ Methods
✔ Constructors

Note:
Classes themselves cannot be protected.

protected Variables

A variable declared as protected can be accessed:

✔ Within the same package
✔ In subclasses outside the package

Example

class Parent {
  protected int data = 100;
}

Accessing protected Variable (Same Package)

class Test {
  public static void main(String[] args) {
    Parent p = new Parent();
    System.out.println(p.data);
  }
}

Accessing protected Variable (Different Package)

package pack2;

import pack1.Parent;

class Child extends Parent {
  void show() {
    System.out.println(data);
  }
}

✔ Access allowed because Child extends Parent

protected Methods

Protected methods follow the same access rules as protected variables.

Example

class Parent {
  protected void display() {
    System.out.println("Protected Method");
  }
}

protected Constructor

A protected constructor can be used to restrict object creation while allowing inheritance.

Example

class Base {
  protected Base() { }
}

✔ Object creation allowed in subclasses
✖ Not allowed from unrelated classes

protected and Inheritance

Protected members are inherited by child classes.

✔ Accessible using inheritance
✔ Supports code reuse

protected vs Other Access Modifiers

✔ private → Same class only
✔ default → Same package only
✔ protected → Same package + subclass
✔ public → Anywhere

Advantages of protected

✔ Supports inheritance
✔ Provides controlled access
✔ Helps design extensible classes

Real-Time Examples

✔ Framework base classes (Spring, Hibernate)
✔ Template method design pattern
✔ Shared logic in parent classes

Common Mistakes

✔ Trying to access protected members without inheritance
✔ Assuming protected is same as default
✔ Using protected for unrelated classes

protected Access Modifier (Interview / Exam Ready)

The protected access modifier in Java allows access to class members within the same package and in subclasses outside the package. It is mainly used to support inheritance and controlled extensibility.

Class and Object in Java

Class and Object are the foundation of Object-Oriented Programming (OOP) in Java.

Everything in Java revolves around classes and objects.

What is a Class?

A Class is a blueprint or template used to create objects.

It defines:
✔ Variables (data / properties)
✔ Methods (behavior / actions)

A class does not occupy memory by itself.

Example of a Class

class Student {
  int id;
  String name;

  void study() {
    System.out.println("Student is studying");
  }
}

What is an Object?

An Object is a real-world entity created from a class.

An object:
✔ Represents a real entity
✔ Occupies memory
✔ Can access class members

Example of an Object

Student s = new Student();

Here:
✔ Student → Class
✔ s → Object reference

Difference Between Class and Object

✔ Class → Blueprint
✔ Object → Instance of a class

✔ Class → Logical entity
✔ Object → Physical entity

✔ Class → No memory allocation
✔ Object → Memory is allocated

Multiple Objects from One Class

A single class can create multiple objects.

Example:

Student s1 = new Student();
Student s2 = new Student();

Each object has its own copy of variables.

Accessing Class Members Using Object

We use the dot (.) operator to access variables and methods.

Example:

s.id = 101;
s.name = "Lokesh";
s.study();

Real-Time Example

Class → Car
Objects → BMW, Audi, Tesla

Each object has:
✔ Same structure
✔ Different values

Why Class & Object Are Important?

✔ Code reusability
✔ Real-world modeling
✔ Easy maintenance
✔ Supports all OOP features

Common Mistakes

✔ Thinking class occupies memory
✔ Forgetting to create object
✔ Accessing non-static members without object

Class & Object (Interview / Exam Ready)

A class is a blueprint that defines properties and behaviors, while an object is an instance of a class that represents a real-world entity and occupies memory.

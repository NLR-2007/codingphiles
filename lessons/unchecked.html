Unchecked Exceptions in Java

Unchecked Exceptions are exceptions that are not checked at compile time.

They occur at runtime and usually happen due to programming mistakes.

What are Unchecked Exceptions?

Unchecked exceptions are exceptions that the compiler does not force the programmer to handle.

They extend the RuntimeException class.

Why Are They Called Unchecked?

They are called unchecked because:

✔ The compiler does not check them
✔ Program compiles successfully even if not handled

However, they may crash the program at runtime.

Hierarchy of Unchecked Exceptions

Throwable
  ↳ Exception
    ↳ RuntimeException
      ↳ Unchecked Exceptions

Common Unchecked Exceptions

✔ ArithmeticException
✔ NullPointerException
✔ ArrayIndexOutOfBoundsException
✔ NumberFormatException
✔ ClassCastException

ArithmeticException

Occurs when an illegal arithmetic operation is performed.

Example

int a = 10 / 0; // ArithmeticException

NullPointerException

Occurs when trying to access an object that has no reference.

Example

String s = null;
System.out.println(s.length()); // NullPointerException

ArrayIndexOutOfBoundsException

Occurs when accessing an invalid index of an array.

Example

int[] arr = {1, 2, 3};
System.out.println(arr[5]); // Exception

NumberFormatException

Occurs when converting an invalid string into a number.

Example

int n = Integer.parseInt("abc"); // Exception

ClassCastException

Occurs when an object is cast to an incompatible type.

Example

Object o = "Java";
Integer i = (Integer) o; // Exception

Handling Unchecked Exceptions

Unchecked exceptions can be handled using:

✔ try-catch block
✔ Proper validation and logic

Example

try {
  int a = 10 / 0;
} catch (ArithmeticException e) {
  System.out.println("Cannot divide by zero");
}

Unchecked vs Checked Exceptions

✔ Unchecked → Runtime checked
✔ Checked → Compile-time checked

✔ Unchecked → Extend RuntimeException
✔ Checked → Extend Exception

✔ Unchecked → Optional handling
✔ Checked → Mandatory handling

Why Java Allows Unchecked Exceptions?

Java allows unchecked exceptions because:

✔ They represent programming errors
✔ Handling all of them makes code complex
✔ Best handled by fixing logic, not catching

Real-Time Examples

✔ Login system → NullPointerException
✔ Calculator → ArithmeticException
✔ Input parsing → NumberFormatException

Common Mistakes

✔ Ignoring runtime errors
✔ Catching generic Exception everywhere
✔ Not validating input

Unchecked Exceptions (Interview / Exam Ready)

Unchecked exceptions in Java are exceptions that occur at runtime and are not checked by the compiler. They extend the RuntimeException class and usually result from programming errors such as invalid calculations, null references, or incorrect type casting. Handling them is optional, but proper coding practices can prevent them.

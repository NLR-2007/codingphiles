Stack Memory in Java

Stack Memory is a runtime memory area in Java used to store method calls, local variables, and reference variables.

Stack memory follows the LIFO principle (Last In, First Out).

Why do we need Stack Memory?

Stack memory is required to manage method execution and store temporary data during program execution.

Without stack memory:
✔ Method calls cannot be tracked
✔ Local variables cannot be stored
✔ Program execution flow breaks

What is Stored in Stack Memory?

Stack memory stores:

✔ Method call information
✔ Local variables
✔ Reference variables to heap objects
✔ Return addresses

How Stack Memory Works

Each time a method is called, a stack frame is created.

When the method finishes execution, its stack frame is removed automatically.

This process is known as push and pop.

Stack Memory Example

class Test {
  void display() {
    int a = 10;
    System.out.println(a);
  }

  public static void main(String[] args) {
    Test t = new Test();
    t.display();
  }
}

✔ Variable a is stored in stack memory
✔ Reference t is stored in stack memory
✔ Object Test is stored in heap memory

Characteristics of Stack Memory

✔ Stores method calls and local variables
✔ Each thread has its own stack
✔ Faster access compared to heap
✔ Memory is automatically allocated and deallocated
✔ Fixed size

Stack Overflow Error

When stack memory is full, the JVM throws a StackOverflowError.

Common causes:
✔ Infinite recursion
✔ Too many nested method calls

Stack Memory vs Heap Memory

Stack Memory:
✔ Stores local variables and method calls
✔ Faster access
✔ Limited size

Heap Memory:
✔ Stores objects and instance variables
✔ Slower access
✔ Larger size

Advantages of Stack Memory

✔ Very fast memory access
✔ Automatic memory management
✔ No memory leaks

Limitations of Stack Memory

✔ Limited size
✔ Cannot store objects
✔ Stack overflow may occur

Real-Time Use Cases

Stack memory is used for:
✔ Method execution tracking
✔ Function calls
✔ Storing local variables
✔ Managing recursion

Common Mistakes

✔ Infinite recursion
✔ Large local variables
✔ Deeply nested method calls

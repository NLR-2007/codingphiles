Built-in Packages in Java

Built-in Packages are predefined packages provided by Java to perform common programming tasks.

These packages are part of the Java API (Application Programming Interface).

What are Built-in Packages?

Built-in packages contain:
✔ Predefined classes
✔ Interfaces
✔ Utilities and helper methods

They reduce development time and make Java powerful and developer-friendly.

Why Built-in Packages are Important?

Built-in packages:
✔ Avoid writing code from scratch
✔ Provide tested and optimized code
✔ Support platform-independent development
✔ Help build enterprise-level applications

Common Built-in Packages in Java

Some of the most frequently used built-in packages are:

✔ java.lang
✔ java.util
✔ java.io
✔ java.sql
✔ java.net
✔ java.time

java.lang Package

The java.lang package contains fundamental classes required for Java programs.

✔ Automatically imported by Java
✔ No need to write import statement

Common classes:
✔ Object
✔ String
✔ Math
✔ System
✔ Wrapper classes (Integer, Double, etc.)

java.util Package

The java.util package provides utility classes and data structures.

Common uses:
✔ Collections Framework
✔ Date and time utilities
✔ Input handling

Common classes:
✔ ArrayList
✔ HashMap
✔ Scanner
✔ Collections
✔ Date

java.io Package

The java.io package is used for input and output operations.

It supports:
✔ File handling
✔ Byte streams
✔ Character streams

Common classes:
✔ File
✔ FileInputStream
✔ FileOutputStream
✔ BufferedReader
✔ BufferedWriter

java.sql Package

The java.sql package is used for database connectivity using JDBC.

It allows Java programs to interact with databases.

Common interfaces/classes:
✔ Connection
✔ Statement
✔ PreparedStatement
✔ ResultSet

java.net Package

The java.net package supports network programming.

Used for:
✔ Client-server applications
✔ Network communication

Common classes:
✔ URL
✔ Socket
✔ ServerSocket

java.time Package

The java.time package (Java 8+) provides modern date and time APIs.

It is immutable and thread-safe.

Common classes:
✔ LocalDate
✔ LocalTime
✔ LocalDateTime
✔ Period
✔ Duration

Importing Built-in Packages

There are three ways to use built-in packages:

✔ Import specific class
import java.util.Scanner;

✔ Import entire package
import java.util.*;

✔ Use fully qualified class name
java.util.Scanner sc = new java.util.Scanner(System.in);

Advantages of Built-in Packages

✔ Faster development
✔ Reliable and optimized code
✔ Platform independent
✔ Easy maintenance

Real-Time Examples

✔ java.util → Collections in enterprise apps
✔ java.io → File upload/download systems
✔ java.sql → Database-driven applications
✔ java.net → Web servers and APIs

Common Mistakes

✔ Forgetting to import required package
✔ Confusing java.util.Date with java.sql.Date
✔ Overusing wildcard imports

Built-in Packages (Interview / Exam Ready)

Built-in packages in Java are predefined packages provided by the Java API that contain commonly used classes and interfaces. They help developers perform tasks such as file handling, collections, database connectivity, networking, and date-time operations efficiently without writing code from scratch.

Default Constructor in Java

A Default Constructor is a constructor that has no parameters.

It is used to initialize objects with default values.

What is a Default Constructor?

A constructor without any arguments is called a default constructor.

If a class does not define any constructor, Java automatically provides a default constructor.

Automatic Default Constructor (Provided by Java)

If no constructor is written, the Java compiler creates a default constructor.

Example:

class Demo {
  int x;
}

Internally, Java creates:

Demo() { }

This constructor initializes:
✔ int → 0
✔ float → 0.0
✔ boolean → false
✔ Object references → null

User-Defined Default Constructor

A programmer can also explicitly define a default constructor.

Example:

class Student {
  int id;

  Student() {
    id = 100;
  }
}

Here, the constructor initializes the variable id with a custom value.

Calling a Default Constructor

A default constructor is called automatically when an object is created.

Example:

Student s = new Student();

Important Points

✔ Default constructor has no parameters
✔ It initializes objects with default values
✔ It is called automatically
✔ Once a constructor is defined, Java does not create one

When Default Constructor is NOT Provided by Java

If a class has any parameterized constructor, Java will not provide a default constructor.

Example:

class Test {
  Test(int x) { }
}

In this case:
✔ Default constructor is NOT created automatically
✔ You must define it explicitly if needed

Real-Time Example

In real life:
✔ A TV without settings starts with default brightness and volume
✔ A mobile phone starts with default system configuration

Common Mistakes

✔ Assuming default constructor always exists
✔ Forgetting to create default constructor when parameterized one exists
✔ Confusing default constructor with parameterized constructor

Default Constructor (Interview / Exam Ready)

A default constructor in Java is a constructor with no parameters. If no constructor is defined, the compiler provides one automatically to initialize objects with default values. If any constructor is defined by the programmer, the default constructor is not provided automatically.

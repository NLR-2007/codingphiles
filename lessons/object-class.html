Object Class in Java

The Object class is the root class of the Java class hierarchy.

Every class in Java directly or indirectly inherits from the Object class.

Why Object Class is Important?

Because all classes inherit from Object, its methods are available to every Java object.

This enables:
✔ Common behavior for all objects
✔ Runtime polymorphism
✔ Framework-level operations

Object Class Hierarchy

Object
  ↳ All Java Classes

Even if you do not write extends Object, Java adds it automatically.

Common Methods of Object Class

The Object class provides several important methods:

✔ toString()
✔ equals(Object obj)
✔ hashCode()
✔ getClass()
✔ clone()
✔ finalize()
✔ wait(), notify(), notifyAll()

toString() Method

The toString() method returns a string representation of an object.

Default implementation prints:
ClassName@HashCode

Example

class Student { }
Student s = new Student();
System.out.println(s);

Equivalent to:
System.out.println(s.toString());

equals() Method

The equals() method is used to compare two objects for equality.

Default behavior compares memory references.

Example

Student s1 = new Student();
Student s2 = new Student();
System.out.println(s1.equals(s2)); // false

✔ Can be overridden to compare object data

hashCode() Method

The hashCode() method returns a unique integer value for an object.

It is mainly used in:
✔ HashMap
✔ HashSet
✔ Hashtable

Rule:
If equals() is overridden, hashCode() should also be overridden.

getClass() Method

The getClass() method returns the runtime class of an object.

Example

Student s = new Student();
System.out.println(s.getClass().getName());

clone() Method

The clone() method creates a copy of an object.

✔ Class must implement Cloneable interface
✔ Creates shallow copy

finalize() Method

The finalize() method is called before garbage collection.

Used to perform cleanup activities.

(Note: Deprecated in modern Java versions)

wait(), notify(), notifyAll()

These methods are used in multithreading.

✔ wait() → thread waits
✔ notify() → wakes one waiting thread
✔ notifyAll() → wakes all waiting threads

Why Object Class Methods Are Overridden?

Developers often override:
✔ toString() → meaningful output
✔ equals() → logical comparison
✔ hashCode() → collection consistency

Real-Time Examples

✔ toString() → logging object details
✔ equals() → comparing entities (User, Product)
✔ hashCode() → storing objects in HashMap

Common Mistakes

✔ Not overriding hashCode() with equals()
✔ Expecting equals() to compare values by default
✔ Misusing clone() without Cloneable

Object Class (Interview / Exam Ready)

The Object class is the root class of Java. All classes inherit from it either directly or indirectly. It provides common methods such as toString(), equals(), hashCode(), getClass(), and wait/notify methods, which are fundamental for object comparison, hash-based collections, reflection, and multithreading.

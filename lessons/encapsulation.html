Encapsulation in Java

Encapsulation is one of the core Object-Oriented Programming (OOP) concepts.

It is the process of wrapping data (variables) and code (methods) together into a single unit.

What Does Encapsulation Mean?

Encapsulation means:
✔ Hiding internal data
✔ Allowing access only through methods
✔ Protecting object state

In Java, encapsulation is achieved using:
✔ private data members
✔ public getter and setter methods

Why Encapsulation is Needed?

Encapsulation provides:
✔ Data security
✔ Controlled access to variables
✔ Better maintainability
✔ Flexibility to change implementation

How to Achieve Encapsulation?

Steps to implement encapsulation:

1. Declare variables as private
2. Provide public getter and setter methods

Example of Encapsulation

class Student {
  private int age;

  public int getAge() {
    return age;
  }

  public void setAge(int age) {
    if(age > 0) {
      this.age = age;
    }
  }
}

✔ age is hidden from outside access
✔ Validation is applied before setting value

Accessing Encapsulated Data

class Test {
  public static void main(String[] args) {
    Student s = new Student();
    s.setAge(20);
    System.out.println(s.getAge());
  }
}

Advantages of Encapsulation

✔ Data hiding
✔ Increased security
✔ Easy debugging
✔ Code reusability
✔ Improved flexibility

Encapsulation vs Data Hiding

✔ Data Hiding → Achieved using access modifiers
✔ Encapsulation → Combination of data hiding + abstraction

Real-Time Example

✔ Bank Account → balance is private
✔ ATM → access through PIN verification
✔ Employee → salary accessed via methods

Common Mistakes

✔ Making variables public
✔ Not using validation in setters
✔ Exposing internal object state

Encapsulation (Interview / Exam Ready)

Encapsulation in Java is the process of wrapping data and methods into a single unit and restricting direct access to data by making variables private and accessing them through public getter and setter methods. It improves security, maintainability, and flexibility.

try-catch in Java

The try-catch block in Java is used to handle runtime exceptions and prevent program termination.

It allows a program to handle errors gracefully and continue execution.

What is try Block?

The try block contains code that may cause an exception.

✔ Risky code is placed inside try block
✔ Cannot exist alone without catch or finally

Syntax

try {
  // code that may cause exception
}

What is catch Block?

The catch block handles the exception that occurs in the try block.

✔ Executes only if exception occurs
✔ Prevents program crash

Syntax

catch(ExceptionType e) {
  // handling code
}

Basic try-catch Example

class Demo {
  public static void main(String[] args) {
    try {
      int a = 10 / 0;
    } catch (ArithmeticException e) {
      System.out.println("Cannot divide by zero");
    }
    System.out.println("Program continues");
  }
}

Flow of try-catch

✔ If no exception → try block executes completely
✔ If exception occurs → control moves to catch block
✔ After catch → normal program flow continues

Multiple catch Blocks

Multiple catch blocks can be used to handle different exceptions.

Example

try {
  int[] arr = {1, 2, 3};
  System.out.println(arr[5]);
} catch (ArithmeticException e) {
  System.out.println("Arithmetic Error");
} catch (ArrayIndexOutOfBoundsException e) {
  System.out.println("Invalid Array Index");
} catch (Exception e) {
  System.out.println("General Exception");
}

✔ Specific exceptions first
✔ General exception last

try-catch-finally

The finally block always executes whether an exception occurs or not.

Used for:
✔ Closing resources
✔ Cleanup operations

Example

try {
  int a = 10 / 2;
} catch (Exception e) {
  System.out.println("Exception occurred");
} finally {
  System.out.println("Finally block executed");
}

Important Rules of try-catch

✔ try must be followed by catch or finally
✔ Multiple catch blocks allowed
✔ Nested try-catch is possible
✔ Parent exception must be caught after child

Nested try-catch

A try-catch inside another try block is called nested try-catch.

Example

try {
  try {
    int a = 10 / 0;
  } catch (ArithmeticException e) {
    System.out.println("Inner catch");
  }
} catch (Exception e) {
  System.out.println("Outer catch");
}

try-catch vs throws

✔ try-catch → Handles exception immediately
✔ throws → Passes exception to calling method

Best Practices

✔ Catch specific exceptions
✔ Do not use empty catch blocks
✔ Do not catch Exception unnecessarily
✔ Use finally or try-with-resources

Real-Time Examples

✔ ATM → divide by zero handling
✔ File upload → file not found handling
✔ Login system → invalid input handling

Common Mistakes

✔ Catching generic Exception everywhere
✔ Writing code after throw statement
✔ Ignoring exception messages

try-catch (Interview / Exam Ready)

The try-catch block in Java is used to handle runtime exceptions. Code that may cause an exception is placed inside the try block, and the exception is handled in the catch block. This prevents abnormal termination of the program and allows graceful error handling.

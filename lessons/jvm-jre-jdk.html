JVM, JRE, and JDK

JVM, JRE, and JDK are the core components of the Java platform. They work together to develop, run, and execute Java programs.

Understanding the difference between JVM, JRE, and JDK is very important for learning Java.

Overview of JVM, JRE, and JDK

The Java platform consists of three main parts:

✔ JVM (Java Virtual Machine)
✔ JRE (Java Runtime Environment)
✔ JDK (Java Development Kit)

Each component has a specific role in Java program execution.

Java Virtual Machine (JVM)

JVM stands for Java Virtual Machine.

It is an abstract machine that provides a runtime environment to execute Java bytecode.

JVM is responsible for:
✔ Loading bytecode
✔ Verifying bytecode
✔ Executing bytecode
✔ Managing memory

Functions of JVM

✔ Loads .class files into memory
✔ Converts bytecode into machine code
✔ Manages memory using Garbage Collection
✔ Provides security using bytecode verification

JVM is Platform Dependent

JVM is platform dependent, which means different operating systems have different JVM implementations.

However, Java programs remain platform independent because the bytecode is the same for all platforms.

Java Runtime Environment (JRE)

JRE stands for Java Runtime Environment.

It provides the environment required to run Java applications.

JRE contains:
✔ JVM
✔ Core Java class libraries
✔ Supporting files

JRE is used by end users who only want to run Java programs, not develop them.

Components of JRE

✔ Java Virtual Machine (JVM)
✔ Java core libraries (java.lang, java.util, etc.)
✔ Runtime supporting files

Java Development Kit (JDK)

JDK stands for Java Development Kit.

It is used by developers to create, compile, and run Java applications.

JDK contains:
✔ JRE
✔ Java Compiler (javac)
✔ Development tools

Tools Included in JDK

✔ javac – Java compiler
✔ java – Java launcher
✔ jar – Archive tool
✔ javadoc – Documentation generator
✔ jshell – Interactive Java shell

Relationship Between JVM, JRE, and JDK

✔ JVM is a part of JRE
✔ JRE is a part of JDK

JDK → JRE → JVM

This means:
✔ JDK is the superset
✔ JVM is the smallest unit

Difference Between JVM, JRE, and JDK

JVM: Executes bytecode, platform dependent
JRE: Provides runtime environment, used to run programs
JDK: Provides development tools, used to create programs

Who Needs JVM, JRE, and JDK?

✔ Developer needs JDK
✔ User needs JRE
✔ Java program runs inside JVM

Real‑World Example

✔ Writing code → JDK
✔ Running code → JRE
✔ Executing code → JVM

Advantages of JVM, JRE, and JDK

✔ Platform independence
✔ Secure execution
✔ Automatic memory management
✔ Easy development and execution

JVM, JRE, and JDK (Exam Ready)

JVM executes Java bytecode, JRE provides the runtime environment, and JDK provides development tools. Together, they form the Java platform that enables platform‑independent, secure, and efficient Java program execution.

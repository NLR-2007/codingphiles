String vs StringBuffer vs StringBuilder in Java

In Java, String, StringBuffer, and StringBuilder are used to handle text data, but they differ in mutability, performance, and thread safety.

Understanding these differences is very important for writing efficient and industry‑level Java programs.

1. String

String is an immutable class. Once a String object is created, its value cannot be changed.

Example:

String s = "Java";
s.concat(" Programming");
System.out.println(s);

Output:
Java

Explanation:
✔ A new object is created
✔ Original string remains unchanged

Features of String

✔ Immutable
✔ Thread-safe (due to immutability)
✔ Stored in String Constant Pool (literals)

2. StringBuffer

StringBuffer is a mutable class used to modify strings without creating new objects.

It is thread-safe because its methods are synchronized.

Example:

StringBuffer sb = new StringBuffer("Java");
sb.append(" Programming");
System.out.println(sb);

Output:
Java Programming

Features of StringBuffer

✔ Mutable
✔ Thread-safe (synchronized)
✔ Slower than StringBuilder

3. StringBuilder

StringBuilder is similar to StringBuffer, but it is not thread-safe.

Because it is not synchronized, it provides better performance.

Example:

StringBuilder sb = new StringBuilder("Java");
sb.append(" Programming");
System.out.println(sb);

Output:
Java Programming

Features of StringBuilder

✔ Mutable
✔ Not thread-safe
✔ Faster than String and StringBuffer

Key Differences

Mutability

✔ String → Immutable
✔ StringBuffer → Mutable
✔ StringBuilder → Mutable

Thread Safety

✔ String → Thread-safe (immutable)
✔ StringBuffer → Thread-safe (synchronized)
✔ StringBuilder → Not thread-safe

Performance

✔ Fastest → StringBuilder
✔ Medium → StringBuffer
✔ Slowest → String (for frequent modifications)

Memory

✔ String → More objects created on modification
✔ StringBuffer / StringBuilder → Same object reused

Comparison Table (Conceptual)

String → Immutable, Secure, Slower for changes
StringBuffer → Mutable, Thread-safe, Slower than Builder
StringBuilder → Mutable, Fastest, Not thread-safe

When to Use What?

Use String when:
✔ Data does not change
✔ Security is important
✔ Read-only text is required

Use StringBuffer when:
✔ Application is multi-threaded
✔ Thread safety is required
✔ Frequent modifications are needed

Use StringBuilder when:
✔ Application is single-threaded
✔ Performance is critical
✔ Heavy string manipulation is needed

Real-Time Use Cases

✔ String → Usernames, passwords, constants
✔ StringBuffer → Multi-threaded logging systems
✔ StringBuilder → Loops, data formatting, reports

String vs StringBuffer vs StringBuilder (Interview / Exam Ready)

String is immutable and thread-safe, StringBuffer is mutable and thread-safe, and StringBuilder is mutable and not thread-safe. Among them, StringBuilder provides the best performance, while StringBuffer is suitable for multi-threaded environments.

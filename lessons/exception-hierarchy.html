Exception Hierarchy in Java

The Exception Hierarchy in Java represents the structure of all exception classes in a parent‑child relationship.

All exceptions and errors in Java are derived from a single root class called Throwable.

Root Class: Throwable

Throwable is the superclass of all errors and exceptions in Java.

Only objects of classes that extend Throwable can be thrown using the throw keyword.

Throwable Has Two Main Subclasses

✔ Error
✔ Exception

Exception Hierarchy Diagram

Throwable
  ↳ Error
  ↳ Exception
    ↳ RuntimeException
    ↳ Other Checked Exceptions

Error

Error represents serious problems that occur due to system failures.

Errors are:
✔ Not recoverable
✔ Not meant to be handled by programs

Examples of Error

✔ OutOfMemoryError
✔ StackOverflowError
✔ VirtualMachineError

✔ Errors occur due to JVM or hardware issues

Exception

Exception represents conditions that a program can handle.

Exceptions are caused due to:
✔ Logical mistakes
✔ Invalid input
✔ External resources

Types of Exceptions

Exceptions are divided into two categories:

✔ Checked Exceptions
✔ Unchecked Exceptions

Checked Exceptions

Checked exceptions are checked at compile time.

They must be:
✔ Handled using try‑catch
✔ Or declared using throws

Examples

✔ IOException
✔ FileNotFoundException
✔ SQLException
✔ ClassNotFoundException

Unchecked Exceptions

Unchecked exceptions occur at runtime.

They extend the RuntimeException class.

Examples

✔ ArithmeticException
✔ NullPointerException
✔ ArrayIndexOutOfBoundsException
✔ NumberFormatException

✔ Not mandatory to handle

RuntimeException

RuntimeException is a subclass of Exception.

It represents programming mistakes that can be avoided by proper coding.

Examples:
✔ Dividing by zero
✔ Accessing null object
✔ Invalid array index

Custom (User‑Defined) Exceptions

Developers can create their own exceptions by extending:

✔ Exception (checked)
✔ RuntimeException (unchecked)

Example

class InvalidAgeException extends Exception {
  InvalidAgeException(String msg) {
    super(msg);
  }
}

Important Rules in Exception Hierarchy

✔ Child exception cannot be broader than parent
✔ Catch block must follow hierarchy order
✔ Specific exception must be caught before generic

Why Exception Hierarchy is Important?

Exception hierarchy helps:
✔ Organize exceptions logically
✔ Handle specific errors precisely
✔ Support polymorphism in exception handling

Real-Time Examples

✔ File operations → IOException
✔ Database access → SQLException
✔ User input → NumberFormatException

Common Mistakes

✔ Catching Exception instead of specific exception
✔ Ignoring checked exceptions
✔ Misunderstanding Error vs Exception

Exception Hierarchy (Interview / Exam Ready)

The exception hierarchy in Java starts with the Throwable class, which has two subclasses: Error and Exception. Errors represent serious system issues and are not handled, whereas exceptions represent recoverable conditions. Exceptions are further divided into checked and unchecked exceptions, enabling structured and flexible error handling in Java programs.

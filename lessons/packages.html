Packages in Java

A Package in Java is a namespace that groups related classes and interfaces together.

It helps organize large programs into logical units, similar to folders in a file system.

What is a Package?

A package is a collection of:
✔ Classes
✔ Interfaces
✔ Sub-packages

Example:
java.lang
java.util
java.io

Why Do We Need Packages?

Packages are used to:
✔ Organize large applications
✔ Avoid class name conflicts
✔ Improve code readability
✔ Provide access control and security
✔ Support reusability

Types of Packages in Java

Java supports two main types of packages:

✔ Built-in Packages
✔ User-Defined Packages

Built-in Packages

Built-in packages are provided by the Java API (Application Programming Interface).

Common built-in packages:
✔ java.lang → Object, String, Math
✔ java.util → ArrayList, Scanner, Date
✔ java.io → File, InputStream, OutputStream
✔ java.sql → JDBC classes

Note:
The java.lang package is imported automatically.

User-Defined Packages

A user-defined package is created by the programmer to organize application code.

Creating a Package

package mypackage;

Rules:
✔ Must be the first statement in the source file
✔ Written before class definition

Example

package mypack;

public class Test {
  public void show() {
    System.out.println("Welcome to Packages");
  }
}

Directory Structure of Package

Package name determines folder structure.

Example:

package com.company.project;

Creates directories:
com/company/project

Accessing Classes from a Package

There are three ways to access a package:

1. Import a Specific Class

import java.util.Scanner;

2. Import Entire Package

import java.util.*;

3. Fully Qualified Name

java.util.Scanner sc = new java.util.Scanner(System.in);

Sub-Packages

A package inside another package is called a sub-package.

Example:
java.util.concurrent

✔ util → parent package
✔ concurrent → sub-package

Advantages of Packages

✔ Better project structure
✔ Avoids class name duplication
✔ Improves security using access control
✔ Easy maintenance and scalability

Packages and Access Control

Packages work together with access modifiers to control visibility.

✔ private → same class
✔ default → same package
✔ protected → same package + subclass
✔ public → everywhere

Real-Time Examples

✔ Banking app → account, loan, customer packages
✔ Web app → controller, service, repository packages
✔ Enterprise apps → layered architecture

Common Mistakes

✔ Writing package statement after imports
✔ Forgetting directory structure
✔ Overusing default package

Packages (Interview / Exam Ready)

A package in Java is a namespace that groups related classes and interfaces together. It helps organize large applications, avoid naming conflicts, and support access control and code reusability. Packages can be built-in or user-defined.

static Keyword in Java

The static keyword in Java is used to define class‑level members.

A static member belongs to the class, not to individual objects.

Why Do We Need the static Keyword?

Normally, each object has its own copy of instance variables.

The static keyword allows sharing a single copy of a variable or method among all objects of a class.

static Variable

A static variable is shared by all objects of the class.

Example

class Student {
  int id;
  static String college = "ABC College";
}

Here:
✔ id → Instance variable (separate for each object)
✔ college → Static variable (shared)

static Method

A static method belongs to the class and can be called without creating an object.

Example

class Demo {
  static void show() {
    System.out.println("Static Method");
  }
}

Calling static method:

Demo.show();

Rules of static Method

✔ Can access only static members directly
✔ Cannot use this keyword
✔ Called using class name

static Block

A static block is used to initialize static variables.

It is executed once when the class is loaded into memory.

Example

class Demo {
  static {
    System.out.println("Static Block Executed");
  }
}

Order of Execution

✔ Static block → executed first
✔ main() method → executed next
✔ Constructors → executed when object is created

static vs Instance Members

✔ static → Belongs to class
✔ Instance → Belongs to object

✔ static → Single copy shared
✔ Instance → Separate copy per object

Memory Allocation

Static members are stored in the Class Area / Method Area.

They are loaded only once, saving memory.

Real-Time Examples

✔ Math.PI (constant value)
✔ Utility methods (Collections.sort())
✔ Counter variables shared across objects

Common Mistakes

✔ Using this inside static method
✔ Accessing instance variables directly in static method
✔ Creating object just to access static members

static Keyword (Interview / Exam Ready)

The static keyword in Java is used to define class-level variables, methods, and blocks. Static members are shared among all objects, loaded once into memory, and can be accessed using the class name without creating objects.

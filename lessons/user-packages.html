User-Defined Packages in Java

A User-Defined Package is a package created by the programmer to organize application-specific classes and interfaces.

It helps structure large projects in a clean, modular, and maintainable way.

What is a User-Defined Package?

A user-defined package is created using the package keyword.

It allows developers to group related classes under a custom namespace.

Why Do We Need User-Defined Packages?

User-defined packages are used to:
✔ Organize large applications
✔ Avoid class name conflicts
✔ Improve code readability
✔ Support access control and security
✔ Enable reuse of classes

Creating a User-Defined Package

To create a package, use the package statement at the top of the source file.

Syntax:

package packageName;

Rules:
✔ Must be the first statement in the file
✔ Written before imports and class definition

Example: Creating a Package

package mypack;

public class Test {
  public void show() {
    System.out.println("User-defined Package Example");
  }
}

Directory Structure of User-Defined Package

The package name determines the folder structure.

Example:

package com.company.project;

This creates the directory structure:
com/company/project

Compiling User-Defined Package

To compile a package and create directories automatically:

javac -d . Test.java

✔ -d → specifies destination directory
✔ . → current directory

Accessing User-Defined Package

There are three ways to access classes from a user-defined package:

1. Import Specific Class

import mypack.Test;

2. Import Entire Package

import mypack.*;

3. Fully Qualified Name

mypack.Test t = new mypack.Test();

Using Package with Access Modifiers

Access modifiers control visibility across packages:

✔ private → same class only
✔ default → same package only
✔ protected → same package + subclass
✔ public → accessible from anywhere

For cross-package access:
✔ Class and methods must be public

Sub-Packages

A package inside another package is called a sub-package.

Example:

package com.company.project.module;

✔ com.company.project → parent package
✔ module → sub-package

Advantages of User-Defined Packages

✔ Better project organization
✔ Improves maintainability
✔ Enhances security using access control
✔ Supports large-scale development

Real-Time Examples

✔ Banking app → account, loan, customer packages
✔ Web app → controller, service, repository packages
✔ Enterprise apps → layered architecture

Common Mistakes

✔ Writing package statement after imports
✔ Forgetting to compile with -d option
✔ Using default package in large projects

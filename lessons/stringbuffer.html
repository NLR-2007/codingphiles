StringBuffer in Java

StringBuffer is a class in Java used to create mutable (modifiable) strings.

Unlike the String class, StringBuffer allows changes to the string without creating new objects.

Package of StringBuffer

StringBuffer belongs to the java.lang package, so it is available by default.

Why StringBuffer is Needed?

The String class is immutable, which means frequent string modifications create many objects and reduce performance.

StringBuffer solves this problem by allowing in-place modification.

Key Features of StringBuffer

✔ Mutable (content can be changed)
✔ Thread-safe (synchronized)
✔ Suitable for multi-threaded applications

Creating a StringBuffer Object

Example:

StringBuffer sb = new StringBuffer("Java");

Modifying StringBuffer

Example:

StringBuffer sb = new StringBuffer("Java");
sb.append(" Programming");
System.out.println(sb);

Output:
Java Programming

Commonly Used StringBuffer Methods

1. append()

Adds text at the end of the existing string.

Example:

sb.append(" World");

2. insert()

Inserts text at a specified position.

Example:

sb.insert(4, " Language");

3. replace()

Replaces characters between specified indexes.

Example:

sb.replace(0, 4, "Core");

4. delete()

Deletes characters between specified indexes.

Example:

sb.delete(0, 4);

5. reverse()

Reverses the content of the string.

Example:

sb.reverse();

Capacity of StringBuffer

StringBuffer has an internal capacity that grows automatically.

Default capacity = 16 + initial string length

Example:

StringBuffer sb = new StringBuffer();
System.out.println(sb.capacity());

Thread Safety in StringBuffer

StringBuffer methods are synchronized, which makes it thread-safe.

This means only one thread can access the StringBuffer object at a time.

Performance

✔ Safer in multi-threaded environments
✔ Slower than StringBuilder due to synchronization

When to Use StringBuffer?

Use StringBuffer when:
✔ String is frequently modified
✔ Application is multi-threaded
✔ Thread safety is required

String vs StringBuffer

✔ String → Immutable
✔ StringBuffer → Mutable & Thread-safe

Limitations of StringBuffer

✔ Slower than StringBuilder
✔ Overhead due to synchronization

Real-Time Use Cases

StringBuffer is used in:
✔ Multi-threaded applications
✔ Logging systems
✔ Server-side string manipulation

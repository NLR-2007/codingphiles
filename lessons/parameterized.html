Parameterized Constructor in Java

A Parameterized Constructor is a constructor that accepts one or more parameters.

It is used to initialize objects with custom values at the time of object creation.

Why Parameterized Constructor?

Default constructors initialize objects with default values.

Parameterized constructors allow:
✔ Passing values during object creation
✔ Initializing objects with meaningful data
✔ Better control over object state

Syntax of Parameterized Constructor

class ClassName {
  ClassName(dataType variable) {
    // initialization code
  }
}

Example of Parameterized Constructor

class Student {
  int id;
  String name;

  Student(int i, String n) {
    id = i;
    name = n;
  }
}

Student s1 = new Student(101, "Lokesh");
Student s2 = new Student(102, "Yash");

Here, each object gets its own values.

Using this Keyword in Parameterized Constructor

The this keyword is used when instance variables and parameters have the same name.

Example:

class Student {
  int id;

  Student(int id) {
    this.id = id;
  }
}

✔ this.id → instance variable
✔ id → constructor parameter

Calling a Parameterized Constructor

A parameterized constructor is called automatically when an object is created with arguments.

Example:

Student s = new Student(101);

Multiple Parameterized Constructors

A class can have multiple parameterized constructors with different parameters.

This is called Constructor Overloading.

Example:

class Demo {
  Demo(int x) { }
  Demo(String s) { }
  Demo(int x, int y) { }
}

Difference Between Default and Parameterized Constructor

✔ Default → No parameters, default values
✔ Parameterized → Takes parameters, custom values

Real-Time Example

✔ Bank Account → accountNumber, balance
✔ Student → id, name
✔ Employee → empId, salary

Common Mistakes

✔ Forgetting to pass arguments while creating object
✔ Assuming Java provides default constructor when parameterized exists
✔ Not using this keyword when needed

Parameterized Constructor (Interview / Exam Ready)

A parameterized constructor in Java is a constructor that accepts parameters and initializes objects with user-defined values at the time of object creation. It provides better control over object initialization.

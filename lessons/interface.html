Interface in Java

An Interface in Java is a blueprint that defines a set of abstract methods that a class must implement.

It is used to achieve full abstraction and multiple inheritance in Java.

What is an Interface?

An interface specifies what a class must do, but not how it does it.

It represents a contract between the class and the outside world.

Why Do We Need Interface?

Interfaces are used to:
✔ Achieve 100% abstraction
✔ Support multiple inheritance
✔ Promote loose coupling
✔ Improve code flexibility and scalability

Syntax of Interface

interface InterfaceName {
  void method1();
  void method2();
}

Implementing an Interface

A class implements an interface using the implements keyword.

Example

interface Vehicle {
  void start();
}

class Car implements Vehicle {
  public void start() {
    System.out.println("Car starts with key");
  }
}

✔ Interface method implemented in class
✔ Method must be public

Multiple Inheritance Using Interface

Java does not support multiple inheritance using classes, but it supports it using interfaces.

Example

interface A {
  void show();
}

interface B {
  void display();
}

class Demo implements A, B {
  public void show() { }
  public void display() { }
}

Interface Variables

All variables in an interface are:
✔ public
✔ static
✔ final

Example:

interface Test {
  int x = 10;
}

Equivalent to:
public static final int x = 10;

Interface Methods (Java 8+)

From Java 8 onwards, interfaces can have:
✔ Default methods
✔ Static methods

Default Method

interface MyInterface {
  default void show() {
    System.out.println("Default Method");
  }
}

Static Method

interface MyInterface {
  static void display() {
    System.out.println("Static Method");
  }
}

Interface vs Abstract Class

✔ Interface → Supports multiple inheritance
✔ Abstract Class → Does not support multiple inheritance

✔ Interface → All methods abstract (before Java 8)
✔ Abstract Class → Can have concrete methods

✔ Interface → Variables are public static final
✔ Abstract Class → Instance variables allowed

Real-Time Examples

✔ Payment → UPI, Card, NetBanking
✔ Database → MySQL, Oracle, MongoDB
✔ Notification → Email, SMS, Push

Common Mistakes

✔ Forgetting to implement all interface methods
✔ Using interface like a class
✔ Reducing method visibility (must be public)

Interface (Interview / Exam Ready)

An interface in Java is a blueprint that contains abstract methods and static final variables. It is used to achieve abstraction and multiple inheritance. Classes implement interfaces using the implements keyword, providing concrete implementations of methods.

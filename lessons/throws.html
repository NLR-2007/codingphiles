throws Keyword in Java

The throws keyword in Java is used to declare exceptions that a method may pass to the calling method.

It does not handle exceptions, it only informs the caller about them.

What is throws?

The throws keyword is used in the method signature to declare exceptions that may occur during execution.

It is mainly used for:
✔ Checked exceptions
✔ Exception propagation

Syntax of throws

returnType methodName() throws ExceptionType {
  // code
}

Simple Example of throws

import java.io.FileReader;

class Demo {
  void readFile() throws FileNotFoundException {
    FileReader fr = new FileReader("data.txt");
  }
}

✔ Compiler allows code
✔ Responsibility passed to caller

Handling Method That Uses throws

The calling method must:

✔ Handle exception using try-catch
✔ OR declare exception using throws

Example

class Test {
  public static void main(String[] args) {
    try {
      Demo d = new Demo();
      d.readFile();
    } catch (FileNotFoundException e) {
      System.out.println("File not found");
    }
  }
}

throws with Multiple Exceptions

A method can declare multiple exceptions using comma-separated list.

Example:

void process() throws IOException, SQLException {
  // code
}

throws with Checked Exceptions

Checked exceptions must be:

✔ Handled using try-catch
✔ OR declared using throws

Otherwise, compile-time error occurs.

throws with Unchecked Exceptions

Declaring unchecked exceptions using throws is optional.

Example:

void calc() throws ArithmeticException {
  int a = 10 / 0;
}

✔ Compiles even without throws

throws vs try-catch

✔ throws → Passes exception to caller
✔ try-catch → Handles exception immediately

✔ throws → Cleaner code
✔ try-catch → More control over handling

throws vs throw

✔ throw → Used to explicitly throw exception
✔ throws → Used to declare exception

✔ throw → Inside method body
✔ throws → In method signature

✔ throw → One exception at a time
✔ throws → Multiple exceptions possible

Exception Propagation using throws

When a method declares an exception using throws, the exception propagates up the call stack.

Example:

method1() throws IOException
  ↳ method2() throws IOException
    ↳ main() handles exception

Real-Time Examples

✔ File handling → IOException
✔ Database operations → SQLException
✔ Network operations → IOException

Common Mistakes

✔ Forgetting to handle method that uses throws
✔ Using throws everywhere unnecessarily
✔ Confusing throws with throw

throws Keyword (Interview / Exam Ready)

The throws keyword in Java is used to declare exceptions that a method may pass to the calling method. It is mainly used for checked exceptions and supports exception propagation, allowing the caller to decide how to handle the exception.
